3Couple signup & first load

A couple signs up once and is sent to their own private URL, which both partners can open on any device.

sequenceDiagram
  actor C as Couple
  participant APP as App (browser)
  participant DB as Supabase
  C->>APP: Enter bride + groom names, date
  APP->>DB: wp_signup(names, date)
  DB-->>APP: { token, rsvp_token }
  APP->>APP: Save token, redirect to #w=token
  APP->>DB: wp_load(token)
  DB-->>APP: names, date, plan, rsvp_token
  APP-->>C: Show planner (checklist, budget, events, guests)
  Note over C,APP: Couple shares the #w link with their partner

4Cross-device live sync

Edits save to the cloud (debounced); other devices poll every few seconds and adopt newer data. Last write wins.

sequenceDiagram
  participant A as Device A (her phone)
  participant DB as Supabase
  participant B as Device B (his phone)
  A->>A: Edit a task / guest / budget
  A->>DB: wp_save(token, data)  (debounced)
  loop every ~7 seconds
    B->>DB: wp_load(token)
    DB-->>B: data + updated_at
    alt updated_at is newer
      B->>B: Adopt data, re-render
    end
  end

5Guest RSVP

The couple shares a separate public RSVP link. Guests reply themselves and it flows into the couple's guest list — guests never see the planner.

sequenceDiagram
  actor Cpl as Couple
  actor Gst as Guest
  participant APP as App
  participant DB as Supabase
  Cpl->>APP: Guests tab -> "RSVP link"
  APP-->>Cpl: yoursite/#rsvp=RSVP_TOKEN
  Cpl-->>Gst: Share RSVP link
  Gst->>APP: Open #rsvp link
  APP->>DB: wp_rsvp_info(rsvp_token)
  DB-->>APP: bride + groom + date
  APP-->>Gst: "You're invited" form
  Gst->>APP: name, attending, party, meal
  APP->>DB: wp_rsvp_submit(...)
  DB->>DB: Append guest to plan
  DB-->>APP: ok -> "Thank you!"
  Note over Cpl,DB: Couple's planner poll picks up the guest

6Admin / master view

Key-protected. The admin signs in once, sees every couple, can open any planner, or delete a couple (with a warning).

sequenceDiagram
  actor Adm as Admin
  participant APP as App
  participant DB as Supabase
  Adm->>APP: Open /#admin
  APP-->>Adm: Admin sign-in (key)
  Adm->>APP: Enter admin key
  APP->>DB: wp_admin_list(key)
  alt key valid
    DB-->>APP: All couples + planner links
    APP-->>Adm: Master table
    opt Delete a couple
      Adm->>APP: Click delete, confirm warning
      APP->>DB: wp_admin_delete(key, token)
      DB-->>APP: ok -> refresh
    end
  else key invalid
    DB-->>APP: unauthorized
    APP-->>Adm: "Access denied"
  end
